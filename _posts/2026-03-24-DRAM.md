# 🧠 SMT ইন্ডাস্ট্রিয়াল মেশিনে DRAM মেমরি পুলিং

আধুনিক SMT (Surface Mount Technology) মেশিনগুলো **real-time industrial software** দিয়ে চলে, যেখানে মেমরি ব্যবহারের আচরণ হতে হয় **predictable, fast, এবং safe**। এজন্য সাধারণ C++ এর `new/delete` ব্যবহার না করে, এসব সিস্টেমে ব্যবহার করা হয় **DRAM memory pooling**।

---

## 🔧 DRAM Memory Pooling কী?

সহজভাবে বললে:

👉 সিস্টেম চালু হওয়ার সময় DRAM (RAM)-এর একটি নির্দিষ্ট অংশ reserve করা হয়
👉 এবং সব dynamic object (module, data structure ইত্যাদি) ওই pool-এর ভেতরেই তৈরি হয়

---

## 📦 উদাহরণ: DRAM Pool Definition

ইন্ডাস্ট্রিয়াল সিস্টেমে সাধারণত এরকম macro দেখা যায়:

```cpp
#define AREA_DRAM_SIZE_TYPE25   0xEC0000   // Size: ~14.75MB (rev B240_8055)
#define DRAM_POOL_MAIN_SIZE     0x01000000 // 16MB main pool
#define DRAM_POOL_MODULE_SIZE   0x00400000 // 4MB modules-এর জন্য
#define DRAM_POOL_BUFFER_SIZE   0x00200000 // 2MB temporary buffer-এর জন্য
```

👉 এগুলো define করে DRAM-এর কতটুকু অংশ কোন কাজে ব্যবহার হবে

---

## 🧠 ভেতরে কীভাবে কাজ করে

সিস্টেম start হওয়ার সময়:

```cpp
void* dram_pool = malloc(DRAM_POOL_MAIN_SIZE);
```

শুরুতে মেমরি অবস্থা:

```
[ free ][ free ][ free ][ free ]  → পুরো pool ফাঁকা
```

---

## ⚙️ Object Allocation (SMT Module)

যখন কোনো module ব্যবহার করা হয় (যেমন Nozzle Adjustment, Calibration):

```cpp
MyModule* obj = new(pcDRamAlloc) MyModule();
```

👉 এর মানে:

* মেমরি নেওয়া হচ্ছে pre-allocated DRAM pool থেকে
* ওই জায়গায় constructor run হচ্ছে

---

### Runtime অবস্থার উদাহরণ:

```
[ used(MainObj) ][ used(NozzleAdj) ][ free ][ used(Buffer) ][ free ]
```

👉 গুরুত্বপূর্ণ:

* ❌ module অনুযায়ী আগে থেকে fixed memory থাকে না
* ✅ runtime-এ free জায়গা অনুযায়ী allocate হয়

---

## 🔁 Deallocation (পুনরায় ব্যবহার)

যখন module বন্ধ হয়:

```cpp
delete obj;
```

👉 তখন মেমরি হয়:

```
[ free ][ used ][ free ][ used ][ free ]
```

👉 allocator আবার reuse করে → efficient ব্যবহার

---

## 🚀 কেন সাধারণ `new/delete` ব্যবহার করা হয় না?

### ❌ সাধারণ C++:

* OS heap ব্যবহার করে
* timing unpredictable
* fragmentation হতে পারে

---

### ✅ DRAM Pooling:

* ✔️ allocation time predictable
* ✔️ fragmentation কম বা controlled
* ✔️ real-time safe
* ✔️ সম্পূর্ণ control

---

## ⚡ Industrial Design Insight

SMT মেশিনে:

* মেমরি নিজেই system architecture-এর অংশ
* যেমন module:

  * Placement control
  * Nozzle adjustment
  * Vision calibration

👉 সবই একই DRAM pool ব্যবহার করে
👉 কিন্তু দরকার হলে তবেই allocate হয়

---

## 🧾 সংক্ষিপ্ত সারাংশ

👉 SMT মেশিনে DRAM-এর একটি নির্দিষ্ট অংশ আগে থেকেই reserve করা থাকে
👉 সব object runtime-এ ওই pool থেকে allocate হয়
👉 কোনো module-এর জন্য fixed address থাকে না
👉 allocator free জায়গা দেখে allocate করে
👉 ফলে system হয় fast, predictable, এবং safe
